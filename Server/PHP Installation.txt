使用
https://webtatic.com/packages/php70/

这个方法可以保证安装上最新的PHP

# groupadd nginx
# useradd -g nginx nginx


1. 使用第三方的源

# yum -y install epel-release
# rpm -Uvh https://mirror.webtatic.com/yum/el7/webtatic-release.rpm

2. 安装

# yum -y install php71w-fpm php71w-cli php71w-gd php71w-mcrypt php71w-mysql php71w-pear php71w-xml php71w-mbstring php71w-pdo php71w-json php71w-pecl-apcu php71w-pecl-apcu-devel


安装完成后，可以使用命令查看安装的版本
# php -v

如果是老版本，可以如下方式更新
# yum install yum-plugin-replace
# yum replace php-common --replace-with=php71w-common

3. 修改php-fpm配置文件

# vi /etc/php-fpm.d/www.conf

查找listen = 127.0.0.1:9000，修改为以下内容
listen = /var/run/php-fpm/php-fpm.sock

listen.owner = nginx
listen.group = nginx

查找user = apache，修改为以下内容
user = nginx

查找group = apache，修改为以下内容
group = nginx

4 启动php-fpm并设置为开机自启

systemctl enable php-fpm.service
systemctl start php-fpm.service

Nigix使用PHP的配置
server节点里面，配置如下：
        location ~ \.php$ {
            try_files  $uri =404;
            root           /data/www;
            fastcgi_pass   unix:/var/run/php-fpm/php-fpm.sock;
            fastcgi_index  index.php;
            fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
            include        fastcgi_params;
        }



No Input File Specifed.
是因为SELinux的权限问题
只需要设置一下，让httpd对新的www的目录具有访问权限就可以，如下：

# chcon -Rt httpd_sys_content_t /data/www
# chcon -Rt httpd_sys_content_rw_t /data/www
